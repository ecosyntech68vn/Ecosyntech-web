# Twitter Auto Post via Shizuku

Execute this deterministic workflow end-to-end unless blocked.

1. Validate prerequisites.
- Confirm `shizuku` command is available.
- Confirm Twitter package exists with `shizuku pm list packages | rg -i com.twitter.android`.
- If missing, stop and report the blocker.

2. Start compose intent with user text.
- Run:
```bash
shizuku am start -a android.intent.action.SEND -t text/plain --es android.intent.extra.TEXT "<TEXT>" -p com.twitter.android
```
- Text with spaces requires the fixed AnyClaw `shizuku` wrapper that shell-quotes each argument. If `am start` reports `pkg=Twitter` instead of `pkg=com.twitter.android`, the wrapper is stale and must be regenerated by restarting/updating AnyClaw.

3. Handle Android share chooser automatically.
- Dump UI: `shizuku uiautomator dump /sdcard/uidump.xml`.
- If package is `com.android.intentresolver`, locate app tile with text `Post` and tap center of its bounds using `shizuku input tap X Y`.
- If chooser shows buttons like `Once`/`Always`, still pick `Post` tile first.

4. Confirm Twitter composer screen.
- Dump UI again and verify package `com.twitter.android`.
- Verify tweet text field exactly matches the intended text (node id `com.twitter.android:id/tweet_text`).
- Do not post if the field only contains the intended text as a substring. On tested X/Twitter builds, `uiautomator` can report the field as `X<TEXT>` after share-intent compose. Treat that as a mismatch unless you correct it and verify again.
- Locate post button (`com.twitter.android:id/button_tweet` or visible text `POST`) and tap center only after exact text verification passes.

5. Post-submission verification.
- Wait 2-3 seconds.
- Dump UI again.
- Report whether compose screen closed and what foreground package is now.
- State clearly if post status is probable vs confirmed.

6. Reporting format.
- Include the exact commands used.
- Include key UI checks discovered in dumps: package names, target node ids/text, and tap coordinates.
- If any step fails, stop immediately and report the precise failing step.

## Guardrails

- Do not post if the text is empty.
- Do not post multiple times unless user explicitly requests retries.
- Do not modify account settings or permissions.
- Keep actions limited to composing and posting one message.
- Before tapping `POST`, final composer text must exactly equal the requested text. If `tweet_text` reports `X` + requested text, stop or correct the field and re-verify; never post the mismatched text.

## Workarounds and Notes

- If Android-side Codex hits an account usage-limit/auth error but desktop Codex works, copy the current desktop auth to Android proot as `/root/.codex/auth.json`, keep a backup of the previous Android auth, and set mode `0600`. Do not print auth contents.
- `scp` to the proot SSH server can close the connection. Use SSH stdin transfer instead: `ssh ... "cat > /root/.codex/auth.json" < ~/.codex/auth.json`.
- Long inline shell/Python/regex commands are fragile through host shell -> SSH -> Android shell. Prefer a temporary script or base64-transferred script for complex probes.
- On macOS, use `base64 -i file` or `base64 < file`; Linux-style `base64 file` is not portable.
- `dumpsys window` can fail to produce a useful foreground parse. Prefer `shizuku dumpsys activity activities` and read `mResumedActivity` / `topResumedActivity`.
- For test cleanup, force-stop Twitter after draft deletion if you need a clean process list: `shizuku am force-stop com.twitter.android`.

## Safe dry-run (no posting)

Use this when the user asks to test the skill without publishing:

1. Run the same compose intent with dry-run text:
```bash
shizuku am start -a android.intent.action.SEND -t text/plain --es android.intent.extra.TEXT "AnyClaw Twitter skill dry run - do not post" -p com.twitter.android
```

2. If Android resolver appears, tap the `Post` tile, then tap `Once`. On the tested POCO device, the observed centers were approximately:
```bash
shizuku uiautomator dump /sdcard/twitter_resolver.xml
python3 - <<'PY'
import re, subprocess
xml = subprocess.check_output(["shizuku", "cat", "/sdcard/twitter_resolver.xml"], text=True)
for label in ("Post", "Once"):
    m = re.search(r'text="' + re.escape(label) + r'"[^>]*bounds="\[(\d+),(\d+)\]\[(\d+),(\d+)\]"', xml)
    if not m:
        print(f"MISSING:{label}")
        continue
    x1,y1,x2,y2 = map(int, m.groups())
    x,y = (x1+x2)//2, (y1+y2)//2
    print(f"TAP:{label}:{x},{y}")
    subprocess.run(["shizuku", "input", "tap", str(x), str(y)], check=True)
PY
```

3. Verify composer but do not tap `POST`:
```bash
shizuku uiautomator dump /sdcard/twitter_dryrun.xml
shizuku cat /sdcard/twitter_dryrun.xml | tr '>' '\n' | grep -E 'package="com.twitter.android"|tweet_text|button_tweet|POST|AnyClaw Twitter skill dry run'
```

For real posting, extract and compare the composer text exactly before tapping `POST`:

```bash
export EXPECTED="AnyClaw Twitter skill dry run - do not post"
shizuku uiautomator dump /sdcard/twitter_dryrun.xml >/dev/null
python3 - <<'PY'
import html
import os
import re
import subprocess
import sys

expected = os.environ["EXPECTED"]
xml = subprocess.check_output(["shizuku", "cat", "/sdcard/twitter_dryrun.xml"], text=True)
node = ""
for m in re.finditer(r"<node [^>]*>", xml):
    if 'resource-id="com.twitter.android:id/tweet_text"' in m.group(0):
        node = m.group(0)
        break
text_match = re.search(r'text="([^"]*)"', node)
actual = html.unescape(text_match.group(1)) if text_match else ""
print("COMPOSER_TEXT=" + actual)
if actual != expected:
    print("TEXT_MISMATCH: refusing to post", file=sys.stderr)
    sys.exit(1)
print("TEXT_OK")
PY
```

4. Clean up the draft:
```bash
shizuku input keyevent KEYCODE_BACK
python3 - <<'PY'
import re, subprocess, sys, time

m = None
xml = ""
for _ in range(10):
    subprocess.run(["shizuku", "uiautomator", "dump", "/sdcard/twitter_cleanup.xml"], check=True, stdout=subprocess.DEVNULL)
    xml = subprocess.check_output(["shizuku", "cat", "/sdcard/twitter_cleanup.xml"], text=True)
    m = re.search(r'text="Delete"[^>]*bounds="\[(\d+),(\d+)\]\[(\d+),(\d+)\]"', xml)
    if m:
        break
    time.sleep(0.5)
if not m:
    print("MISSING:Delete")
    print(xml[:1000])
    sys.exit(1)
x1,y1,x2,y2 = map(int, m.groups())
x,y = (x1+x2)//2, (y1+y2)//2
print(f"TAP:Delete:{x},{y}")
subprocess.run(["shizuku", "input", "tap", str(x), str(y)], check=True)
PY
```

Always verify the final focus is no longer the composer or that the composer text is gone.
If the composer was not open, missing `Delete` is not necessarily an error; first check whether `tweet_text` or `button_tweet` still exists. If neither exists and foreground is AnyClaw, cleanup is already complete.
